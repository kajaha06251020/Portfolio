# FF5/FF6 バトルシステム仕様リファレンス

## 目的
この資料は、FF5/FF6系の戦闘システムを実装するための設計基準を整理したものです。特に以下を実装判断の基準にします。

- ダメージ計算（物理・魔法）
- レベルアップとステータス成長
- ジョブ/アビリティ/魔石/召喚/魔法
- 効果範囲・消費コスト
- 装備制限・状態異常・ATBなどの戦闘ルール

---

## 1. ダメージ計算式

### 1.1 魔法ダメージ（FF6ベース）
基本形：

- ダメージ = 呪文パワー × 4 + (レベル × 呪文パワー × 魔力 / 32)

主要な補正：

- 装備補正（例：イヤリング、勇者の指輪）: ×1.25
- 位置補正（背面など）: ×0.5 など
- 乱数補正: 約 87.5% ～ 100%
- 属性補正:
  - 弱点: ×2.0
  - 耐性: ×0.5
  - 無効: 0
- 最終段で対象の魔法防御により軽減

推奨適用順（実装順）：

1. 基礎値を算出
2. 装備・姿勢などの攻撃側補正
3. 乱数補正
4. 対象の魔法防御軽減
5. 属性補正
6. 下限/上限クランプ（例：0～9999）

### 1.2 物理ダメージ（FF6ベース）
基本定義：

- Attack = 武器攻撃力 + 2 × Vigor（Vigor ≥ 128 なら Attack は 255 固定）
- 基礎ダメージ = 武器攻撃力 + ((レベル² × Attack) / 256) × 1.5

主要な補正：

- クリティカル（約 1/32）: ×2
- バーサク: ×1.5
- 乱数補正: 約 87.5% ～ 100%
- 対象防御で軽減

推奨適用順（実装順）：

1. Attack と基礎ダメージ算出
2. クリティカル/バーサク等の攻撃側補正
3. 乱数補正
4. 防御軽減
5. 下限/上限クランプ

### 1.3 物理/魔法の統一抽象（FF5系の設計しやすい形）
FF5寄りの汎用表現：

- ダメージ = max(0, A - D) × M

定義：

- A: 攻撃パラメータ（物理なら武器＋力、魔法なら呪文パワー等）
- D: 防御パラメータ（防御力/魔法防御）
- M: レベル/能力値依存の倍率

最終段で以下を追加適用：

- クリティカル、バーサク、前列/後列、背面、乱数、属性、状態補正
- 最大ダメージ上限（例：9999）

## 2. ステータス成長・レベルアップ

### 2.1 FF6の成長
- STR/AGI/VIT/MAG などの基礎値はキャラ固有で固定。
- レベルアップで自動上昇する主対象は HP/MP。
- STR等の恒久強化は主に装備や魔石（エスパー）由来。
- 魔石装備時のレベルアップで恒久ボーナス付与（例：STR+1、HP獲得量+30%）。

実装上の要点：

- 「レベルアップ処理」と「魔石ボーナス処理」を分離する。
- レベルアップ時、装備中魔石があれば追加成長テーブルを適用。

### 2.2 FF5の成長
- ジョブ/キャラごとにベース能力が定義され、レベルアップでランダム成長しない。
- HP/MPは固定テーブル（倍率/増加量）で管理。
- ジョブ補正や習得能力は基本値へ加算する構造。
- 能力強化アイテムで任意に伸ばす導線あり。

実装上の要点：

- レベル成長テーブルをデータ化（JSON/CSV推奨）。
- ステータス算出を「基礎値 + ジョブ補正 + 装備補正 + 恒久補正」で統一。

## 3. ジョブ・アビリティ・魔石・召喚・魔法

### 3.1 FF5 ジョブシステム
- 戦闘後にジョブ変更可能。
- ABPでジョブレベルを上げ、アビリティを習得。
- 習得アビリティは他ジョブで装備可能（継承）。
- ジョブマスター時ボーナスがフリーランサー/すっぴん系へ永続継承。

実装上の要点：

- `Job`, `Ability`, `ABPProgress` を分離したデータモデルにする。
- 「現在ジョブ」と「装備アビリティ」を独立管理する。

### 3.2 特殊ジョブ/アビリティ
- 青魔法: 敵技を受ける等の条件で習得。
- ダンサー/モンク等: 固有コマンド・発動条件あり。
- 召喚士: 召喚コマンド利用。

実装上の要点：

- 習得条件をイベントフック化（被弾時、戦闘終了時など）。
- コマンド可視性をジョブ/装備アビリティで判定。

### 3.3 FF6 魔石システム
- 魔石ごとに召喚が紐づく（戦闘中1回）。
- 魔石装備で対象魔法をAP学習可能。
- 学習率 × 獲得AP を累積し、100%到達で習得。
- レベルアップ時に魔石固有ボーナスを付与。

実装上の要点：

- `EsperEquip`（装備中魔石）
- `SpellLearning`（魔法ごとの習得率）
- `SummonUsage`（戦闘中1回制限）

を別管理にする。

## 4. 魔法・アビリティの効果範囲と消費コスト

### 4.1 コスト管理
- 魔法は固定MP消費。
- 召喚は高コスト傾向 + 戦闘中1回制限。
- 一部アビリティはMPでなく回数消費（投げ、特殊舞など）。

実装上の要点：

- `cost_type` を `MP` / `Count` / `Item` で分ける。
- 同一UI上でコスト表示を統一（例：`MP 8`, `残回数 2`）。

### 4.2 効果範囲
- 単体/全体/ランダム複数/味方全体などを技ごとに定義。
- 上位魔法・召喚は全体対象が多い。
- 支援・回復・状態回復も範囲指定を持つ。

実装上の要点：

- `target_scope` をデータ定義する。
- ターゲット選択UIは `scope` に応じて分岐。

## 5. その他の代表的システム

### 5.1 装備制限
- ジョブ（FF5）/キャラ（FF6）で装備可能種別を制限。
- 役割設計（重装/軽装/ローブ等）と強く連動。

実装上の要点：

- `equip_rules` をジョブ/キャラごとにデータ化。

### 5.2 状態異常
- 毒、睡眠、沈黙、暗闇、混乱、麻痺、石化、即死等。
- 治療魔法/アイテム/耐性装備で対処。

実装上の要点：

- 状態異常を「行動不能系」「継続ダメージ系」「命中率系」などで分類。
- ターン更新順（継続ダメージ処理順）を固定ルール化。

### 5.3 戦闘コマンド
- 基本コマンド：たたかう、まほう、アビリティ、アイテム、にげる。
- FF5: ジョブ依存でコマンド増減。
- FF6: キャラ固有コマンド（例：魔封剣など）。

実装上の要点：

- コマンド生成を固定配列でなく「ユニット状態から動的生成」にする。

### 5.4 ATB（アクティブタイムバトル）
- 素早さ + 装備重量などでゲージ速度が変動。
- ゲージ満タンでコマンド入力可能。
- 行動中も他ユニットのATBが進行（ウェイト/アクティブ設定で調整）。

実装上の要点：

- `atb_speed = f(agi, weight, haste/slow)` を1フレーム更新。
- 入力待ち中の時間進行モード（Active/Wait）をオプション化。

## 6. 実装向け推奨データモデル（簡易）

### 6.1 戦闘ユニット
- name
- level
- hp, mp, max_hp, max_mp
- str, vig, mag, agi, vit
- attack_power, physical_def, magic_def
- element_affinity（weak/resist/null）
- status_effects
- atb_gauge, atb_speed

### 6.2 行動データ（魔法/技/アイテム共通）
- id, name, category
- power, hit_rate
- element
- target_scope
- cost_type, cost_value
- max_use_per_battle（召喚など）
- flags（critical_allowed, back_attack_bonus等）

### 6.3 成長データ
- level_table_hpmp（FF5型）
- esper_bonus_table（FF6型）
- job_bonus_table
- ability_unlock_table（ABP）

---

## 7. このプロジェクトへ落とし込む優先実装順

1. ダメージ計算モジュール分離（物理/魔法）
2. 属性補正・防御補正・乱数補正の適用順を固定
3. ATB速度に素早さ/装備重量を反映
4. コマンドをデータ駆動化（魔法・アイテム・アビリティ）
5. 魔法/アイテム/召喚のサブ選択UIを統一
6. 成長処理をFF5型/FF6型で切替可能にする

---

## 8. 補足（実装時の注意）

- 最終ダメージは必ず整数化し、下限/上限を統一する。
- 戦闘ログと実ダメージ計算の値を一致させる。
- 複数体戦ではターゲット消滅時の再選択を安全に処理する。
- 将来の敵/味方増加を見据え、描画座標は自動レイアウト関数で算出する。
